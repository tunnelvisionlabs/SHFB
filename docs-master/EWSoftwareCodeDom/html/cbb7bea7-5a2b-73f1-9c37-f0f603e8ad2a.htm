CSharpCodeProviderWithDocs Class

This is a custom C# code provider that is able to output an individual XML comments file for each unit that is compiled to a folder of your choice.

Remarks
This changes the default behavior which overwrites the comments file on each invocation and dumps it into the temporary ASP.NET compilation folder.

A /docpath:[path] option should be added to the compilerOptions attribute in the Web.config file to specify the path to which the XML comments files will be written. The filenames will match the assembly names generated by the compiler.
