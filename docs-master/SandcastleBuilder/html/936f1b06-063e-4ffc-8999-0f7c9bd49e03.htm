Lightweight Table of Contents Plug-In

This plug-in replaces the frames-based HTML Table of Contents in Website output with a "lightweight" table of contents. The files created by other output methods (such as HTML Help output) are not altered.

Advantages of the Lightweight TOC

  • No more use of HTML frames in the documentation, which means the browser's address bar always contains the correct link to the current documentation page.

  • Improved page load performance, especially for large documentation sites.

  • Proper support for sites that only serve static content. The output of this plugin does not require server-side support for scripting languages such as PHP or ASP.

Disadvantages of the Lightweight TOC

  • Switching to the lightweight TOC will break existing links to specific documentation topics from external sites.

  • Offline browsing of the HTML output requires special consideration, as described below.

  • The Lightweight TOC output does not support the index or search features provided with the frames-based table of contents.

Enabling the Lightweight TOC
 Important

The lightweight TOC plug-in is not compatible with all presentation styles. Only the VS2013Alt presentation style is known to be fully compatible with the features of this plug-in.

The title of the root TOC element for the Lightweight TOC is taken from the first page of conceptual content listed in the .content file used by the project. For best results, the first node should be a root content container as described in Content Placement Options.

Offline Browsing

Due to the use of jQuery and AJAX in the Lightweight TOC, offline browsing requires special consideration.

  1. Download a copy of the Website output for the documentation you wish to browse (or use SHFB to build a copy of it locally).

  2. Open Visual Studio. This method is tested with Visual Studio 2005-2013, however Visual Studio 2012 and newer offer improved support for index.html files which may save one step in the process.

  3. Select File → Open → Web Site.... In the Folder text box, enter the path to the documentation you wish to browse. This folder should contain the index.html file created by the Lightweight TOC Plug-In, and have subfolders including html and toc which contain the actual content of the site.

  4. In Solution Explorer, right click the folder containing index.html and select View in Browser.

  5. If a directory listing appears instead of the default page of your site, locate and click index.html to begin browsing the documentation.
